## 動作バージョン

- Nuxt 3.2.3

### ディレクトリ構成とルーティング

- pagesディレクトリ配下の構造でルーティングが決まる
    - 3.7以降は明示的に設定ファイルに以下を記述する必要あり
        
        ```tsx
        // nuxt.config.ts
        export default defineNuxtConfig({
          pages: true
        })
        ```
        
    - app.vue
        
        ```tsx
        <template>
          <div>
            <NuxtPage />
          </div>
        </template>
        
        ```
        
        - <NuxtPage> は Nuxt に付属する組み込みコンポーネント
        - pages/ ディレクトリ内に配置されたトップレベルページやネストされたページを表示できます。
- 内部でVueRouterが動いている
    - pages/index.vue → `/`
    - pages/price.vue → `/price`

## ページ遷移

- `<NuxtLink>`コンポーネントを使用する
    - Ex
        
        ```tsx
        <template>
          <div>
            <h1>Price Page</h1>
            <hr>
            <NuxtLink to="/">Top Page</NuxtLink>
          </div>
        </template>
        ```
        
        アンカータグがレンダリングされる
        
    - 動的ルート
        - Ex. `~/pages/posts/[id].vue。`
            
            ```tsx
            <template>
              <NuxtLink :to="{ name: 'posts-id', params: { id: 123 } }">
                Post 123
              </NuxtLink>
            </template>
            ```
            
            実際のレンダリング `<a href="/posts/123">Post 123</a>`
            
    - 静的ファイル
        
        public配下のファイルの参照はexternalプロパティを使用する
