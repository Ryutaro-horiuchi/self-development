# 4.9.1 メール送信の問題の概要

### メールヘッダ・インジェクション脆弱性

- 詳細は4.9.2 メールヘッダインジェクション にて

### hiddenパラメーターによる宛先保持

- 送信先アドレスをhiddenパラメーターで保持している場合、任意のアドレスに変更され、迷惑メールの送信に悪用される可能性がある

### メールサーバーによる第三者中継

- メールサーバーの設定に問題があると、そのメールサーバーの発信者でもない受信者でもない第三者のメールを中継する場合がある
    - → 迷惑メールの送信に悪用される場合がある
- 最新のメールサーバーソフトでは、デフォルト状態で第三者中継を許さない設定になっている
- 脆弱性診断でも第三者中継チェックできるものがあるため、メールサーバーをセットアップした際は確認しておくと良い

# 4.9.2 メールヘッダインジェクション

## 概要

- 宛先や件名などのメールヘッダを外部から指定する際に、改行文字を使ってメールヘッダや本文を追加・変更する方法

## 攻撃手法と影響

### 前提

- phpの`mb_send_mail`関数
    - 第4引数の引数で、ヘッダを追加できる。改行を使用することで複数のヘッダを指定可能
        
        → 外部からの入力で上記の仕様を理解し、改行に検証を入れないと脆弱性の要因になる
        
- `<input type="text" name="from">`
    - `name=from`の入力値がmb_send_mail関数の第4引数になるとする

### 攻撃

- 宛先の追加
    - `<textarea name="from" rows="4" cols="30"></textarea>`のようなフォームで、改行を活用すると複数のメールアドレス宛に送信できてしまう
    
    !image.png
    
- 本文の改ざん
    - メール欄のFromに1行改行を挟んで本文を記述する
        
        !image.png
        
    - 下記のように本文が書き換えられている
        
        !image.png
        
- 下記のような入力をすると、添付ファイルも送信される
    
    !image.png
    

### 脆弱性が生まれる原因

- メールのメッセージ形式は、ヘッダとボディを空行で区切るというHTTPと似た形式をとっている
    
    !IMG_0933.jpeg
    
- 外部から指定できるパラメーターに改行を入力できると脆弱性につながる

### 対策

- メール送信には専用のライブラリを使用する
- 外部からのパラメーターをメールヘッダに含ませないようにする
- 外部からのパラメーターには改行を含まないようにメール送信時にチェックする
