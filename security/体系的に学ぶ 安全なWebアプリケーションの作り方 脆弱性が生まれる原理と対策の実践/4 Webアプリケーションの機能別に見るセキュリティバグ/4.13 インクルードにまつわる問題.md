# 4.13.1 ファイルインクルード攻撃

## 概要

- 外部からソースの一部を読み込むインクルード機能にまつわる脆弱性
- アプリケーションが意図しないファイルを指定することにより、脆弱となる
    - PHPでは、設定によっては外部サーバーのURLをファイル名として指定できる場合があるこれをリモート・ファイルインクルード(RFI)と呼ぶ

## 攻撃手法と影響

### 前提

- headerクエリパラメーターで渡された名前のファイルを読み込むサイト
    
    ```php
    <body>
    <?php
    	requre_once($headder . '.php')
    <?>
    </body>
    ```
    

### 攻撃

```php
http://example.jp/4d/4d-001.php?header=../../../../../etc/hosts%00
```

- ディレクトリ・トラバーサル攻撃の手法を応用して、上記呼び出すと下記非公開の/etc/hosts/の内容が表示される
    
    !image.png
    

## 脆弱性が生まれる原因

- インクルードファイル名を外部から指定することができる
- インクルードすべきファイル名かどうかの妥当性チェックをしていない

## 対策

- ディレクトリ・トラバーサル攻撃(対策)の対策と同じく、外部からファイル名を指定する仕様を避けること、またファイル名を英数字に限定する
