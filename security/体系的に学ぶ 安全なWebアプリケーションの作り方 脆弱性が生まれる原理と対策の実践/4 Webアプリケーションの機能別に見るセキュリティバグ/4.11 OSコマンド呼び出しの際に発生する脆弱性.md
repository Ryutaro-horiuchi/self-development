# 4.11.1 OSコマンド・インジェクション

## 概要

- Web開発の言語の多くはシェル経由でOSコマンドを呼び出す機能を提供している
- シェルを呼び出せる機能の使い方に問題があると、意図しないOSコマンドが実行可能になる場合がある
    
    → OSコマンド・インジェクション脆弱性につながる
    

## 攻撃手法と影響

- 前提
    - メールアドレスフォームの入力値を`$mail`変数に格納し、下記のようにphpのsystem関数でsemdmailコマンドを呼び出すプログラム
        
        ```bash
        system("/usr/sbin/sendmail -i < template.txt $mail");
        ```
        
- 攻撃
    - メールアドレスフォームに下記入力してOSコマンドインジェクション攻撃を実行してみる
        
        ```
        bob@example.com;cat /etc/passwd
        ```
        
    - /etc/passwdが閲覧できてしまう
        
        !image.png
        
        - 
- この例では表示だけだが、OSコマンド・インジェクション攻撃により、ファイルの削除、変更、外部からのファイルダウンロード、ダウンロードした不正ツールの悪用などWebアプリケーションが稼働するユーザー権限で実行できるコマンドは全て悪用可能

## 脆弱性が生まれる原因

- シェルには複数のコマンドを起動するための構文があり、外部からパラメーターを操作することにより、元のコマンドに加えて、別のコマンドを起動させられる場合がある。
    
    → OSコマンド・インジェクション脆弱性につながる
    
- 以下が主な原因
    - シェル機能を呼び出せる関数(system, openなど)を利用している
    - シェル呼び出しの機能のある関数にパラメーターを渡している
    - パラメーター内に含まれるシェルのメタ文字がエスケープされていない
        - シェルには1行の指定で複数のプログラムを起動する方法が用意されており、これの悪用がOSコマンド・インジェクション
            - Ex.
                
                ```bash
                $ echo aaa ; echo bbb # コマンドを続けて実行
                aaa
                bbb
                
                $ echo aaa && echo bbb # 最初のコマンドが成功したら第2のコマンドを実行 
                ```
                
        - `;` や `&&`など特別な意味を持つ文字をシェルのメタ文字であり、これをエスケープしてあげる必要がある
