## 脆弱性はどこで発生するのか

- 脆弱性は処理に起因するものと出力に起因するものがある
- 入力に起因する脆弱性はない(アプリケーション・セキュリティの文脈において)
- 出力に起因する脆弱性には「インジェクション」という単語がつくものが多い

## インジェクション系脆弱性とは

- 入力のデータの中に引用符やデリミタなど「データの終端」を示すマークを混入させて、その後の文字列の構造を変化させる脆弱性のこと
- Ex. SQLインジェクション
    
    ```python
    SELECT * FROM users WHERE id='$id'
    ```
    
    右記の文字列 ‘`;DELETE FROM users - -`’   を流し込むとする
    
    ```python
    SELECT * FROM users WHERE id='';DELETE FROM users --'
    ```
