# 4.14.1 evalインジェクション

## 概要

- 文字列をスクリプトソースとして解釈して実行するevalと呼ばれる機能
- evalの利用方法に問題がある場合、外部から送り込んだスクリプトを実行される場合がある。
    - この攻撃をevalインジェクション攻撃といい、そのような攻撃を受ける脆弱性をevalインジェクション脆弱性と呼ぶ

## 攻撃手法と影響

### 脆弱なアプリケーションの説明

- 複雑なデータを文字列に変換(シリアライズ)に変換してフォーム間で受け渡しをする場合の脆弱性
    - 入力フォーム一部
        - PHPのvar_exportを使用して式の値をPHPのソースの形式で返す(シリアライズ)
        
        ```php
        ...
        $ex = var_export(array(1,2,3), true)
        $b64 = base64_encode($ex)
        ...
        ```
        
    - 出力
        - evalを用いてデシリアライズ(元のデータに戻す)を行う($b64の値がクエリパラメーターとして渡される)
            
            ```php
            <?php
            	$str = base64_decode($data)
            	eval('$a = ' . $str . ';')
            ?>
            <body>
            <?php var_dump($a); ?>
            ```
            
            !image.png
            
            → 元の配列に戻っている
            

### 攻撃

evalに対して、外部からのパラメーターで任意の式を注入できる

- Ex. 下記を注入するとする
    
    ```php
    $a 0; phpinfo();
    ```
    
    - OWASP ZAPの機能で文字列をBase64エンコード(`MDsgcGhwaW5mbygp)`
    - 攻撃
        
        ```html
        http://example.jp/4e/4e-002.php?data=MDsgcGhwaW5mbygp
        ```
        
        !image.png
        
        phpinfo関数が実行されてしまう。
        

## 脆弱性が生まれる原因

- evalを扱うこと自体がそもそも危険である
- evalに与えるパラメーターのチェックがされていない

## 対策

- eval(同等機能を含む)を使わない
- evalの引数に外部からのパラメーターを使用しない
- evalの与える外部からのパラメータを英数字に制限する

# 4.14.2 安全ではないデシリアライゼーション

## 概要

- アプリケーション内部の構造を持ったデータを保存・伝達する目的でバイト列に変換することをシリアライズ。シリアライズされたデータを復元することをデシリアライズと呼ぶ
- シリアライズされたデータが信頼できない場合、デシリアライズの処理の際に意図しないオブジェクトが生成され、任意のコードを実行されてしまう場合がある
