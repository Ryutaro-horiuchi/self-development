# 4.10.1 ディレクトリ・トラバーサル

## 概要

外部からパラメーターの形でサーバー上のファイル名を指定できるWebアプリケーションでは、ファイル名に対するチェックが不十分であると、アプリケーションの意図しないファイルに対して、閲覧や改ざんができる場合がある

### 脆弱性による影響

- Webサーバー内のファイルの閲覧 → 重要情報の漏洩
- Webサーバー内のファイルの改ざん
    - マルウェアのサイトに誘導する仕組みの書き込み
    - スクリプトファイルや設定ファイル削除によるサーバー機能停止

## 攻撃手法と影響

- 前提
    - templateパラメーターでファイル名を指定できるとする
- 正常系
    - `http://example.jp/4a/4a-001.php?template=spring`
    - 組み立て後のファイル名
        
        `/var/www/htm/4a/tmpl/spring.html`
        
- 攻撃例
    - `http://example.jp/4a/4a-001.php?template=../../../../../etc/hosts%00`
    - 組み立て後のファイル名
        
        `/var/www/htm/4a/tmpl/../../../../etc/hosts[NUL].html`
        
    - 正規化後のファイル
        
        `/etc/hosts`
        
    - 表示されてしまっている
        
        !image.png
        

## 脆弱性が生まれる原因

ディレクトリ・トラバーサル脆弱性が発生する条件は以下3つがある

1. ファイル名を外部から指定できる
2. ファイル名として、絶対パスや相対パスの形で異なるディレクトリを指定できる
3. 組み立てたファイル名に対するアクセスの可否をチェックしていない

## 対策

- 外部からファイル名を指定できる仕様を避ける
    - ファイル名を固定にするなど
- ファイル名にディレクトリ名が含まれないようにする
    - ディレクトリを示す`「/」「\」「:」`が含まれないようにすれば、アプリケーションが想定したディレクトリのみのアクセスとなる
    - PHPの場合はbasenameという関数が使用できる
- ファイル名を英数字に限定する
    - 正規表現
