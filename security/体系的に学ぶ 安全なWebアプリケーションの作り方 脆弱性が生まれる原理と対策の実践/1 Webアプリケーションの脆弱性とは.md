## 脆弱性とは「悪用できるバグ」である

- 悪用の例
    - 個人情報などの秘密情報を勝手に閲覧する
    - サイトを閲覧した利用者のPCをウイルスに感染させる

### 脆弱性があるとなぜダメなのか

- 経済的損失
    - 利用者が受けた金銭的損失の補填・補償
    - Webサイト停止による機会損失
- 法的な要求
- 利用者が回復不能なダメージを受ける場合が多い
    - 一旦漏洩した個人情報を回収することは不可能
    - クレジットカード番号漏洩した場合、補填はできても利用者が受けた不安や精神的苦痛は補填できない
- 攻撃インフラ(ボットネットワークなど)に加担してしまう
    - ボットはマルウェアの一種。PC感染後、外部からの指令を受けて、迷惑メール送信やDDos攻撃などの不正活動を行うもの

### 脆弱性が生まれる理由

1. バグによるもの
    1. SQLインジェクションやXSS
2. チェック機能の不足によるもの

## セキュリティガイドライン

- 安全なウェブサイトの作り方
    - IPAが公開している冊子で、IPAに届出の多い脆弱性を中心に、その解決策と保険的な対策を説明したもの
- OWASP Top 10
    - OWASP(The Open Web Application Security Project)は、Webアプリケーションセキュリティの課題解決を目的とするオープンなコミュニティ
    - 2004年以降、定期的にOWASP Top 10を公表。グローバルな企業を中心に社内のwebアプリケーションセキュリティの基準として活用されているが、最近は日本国内での活用も増えている
