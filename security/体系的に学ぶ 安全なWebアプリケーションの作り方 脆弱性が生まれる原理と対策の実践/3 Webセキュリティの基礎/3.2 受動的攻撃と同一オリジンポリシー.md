## 能動的攻撃と受動的攻撃

- Webアプリケーションに対する攻撃は「能動的攻撃」と「受動的攻撃」に分類される

### 能動的攻撃

- 攻撃者がWebサーバーに対して直接攻撃すること
- Ex. SQLインジェクション

### 受動的攻撃

- Webサイトの利用者に罠を仕掛けることにより、罠を閲覧したユーザーを通してアプリケーションを攻撃する手法のこと
- 単純な受動的攻撃
    
    いわゆる「怪しいサイト」を閲覧してマルウェアに感染する場合
    
- 正規サイトを悪用する受動的攻撃
    - 攻撃者があらかじめ正規サイトに攻撃してコンテンツに仕掛けを仕込む → 利用者が仕掛けを含むコンテンツを閲覧するとマルウェアに感染する
    - Ex. SNSなどの利用者が投稿できるサイト機能のクロスサイト・スクリプティング脆弱性を悪用する
- サイトを跨った受動的攻撃
    - 罠サイトと正規サイトを跨った攻撃
        - 正規サイトにログイン利用しているユーザーのアカウントを悪用した攻撃であることが特徴
    - 手順
        1. 利用者が罠サイトを閲覧する
        2. 罠サイトから、仕掛けを含むHTMLをダウンロードする
        3. HTMLの仕掛けが発動して正規サイトに攻撃のリクエストを送信する
        4. 正規サイトからJavaScriptなどの仕掛けを含むレスポンスが返る
            
            このステップはない場合もある
            

## ブラウザの受動的攻撃対策 同一オリジンポリシー

- JavaScriptなどのクライアントスクリプトからサイトを跨ったアクセスを禁止するセキュリティ上の制限の一つ。
- 異なるホストに置かれたJavaScriptからのアクセスは拒否される
    - iframe内にある別ドメインのデータを読み取ろうとした時
    - AjaxのXMLHttpRequestオブジェクトで、別ドメインにアクセスしようとした時

### 同一オリジンである条件

- URLのホストが一致している
- スキーム(プロトコル)が統一している
- ポート番号が一致している

### JavaScript以外のクロスドメインアクセス

JavaScript以外のブラウザ機能で、クロスドメインが許可されているもの

- frame要素とiframe要素
- ime要素
- script要素
    - src属性を使用して他のサイトからJavaScriptを読み込める
- form要素のaction属性
